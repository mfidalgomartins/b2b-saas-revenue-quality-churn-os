# Dashboard Architecture

## Official Build Path
- Builder: `src/dashboard/build_executive_dashboard.py`
- Output: `outputs/dashboard/executive_dashboard.html`
- Pipeline integration: `src/pipeline/run_project_pipeline.py` (`dashboard` + `dashboard_refresh` after validation)

## Data Model (Embedded JSON)
The dashboard embeds a governed payload in `<script id="dashboard-data" type="application/json">`.

Main payload blocks:
- `meta`: version stamp, readiness tier, data coverage, row counts.
- `official_kpis`: approved KPI snapshot from validated metric outputs.
- `alerts`: threshold-based executive alerts.
- `accounts`: account-level diagnostic table used for filter-driven slicing.
- `manager_panel`: manager attributes/portfolio fields.
- `scenario_cards`, `risk_impact`: forecast and impact layer.
- `chart_catalog`: embedded chart images (base64) sourced from `outputs/charts`.
- `methodology`, `source_map`: explainability and governance traceability.

## Section-to-Source Mapping
- Executive overview: `reports/main_business_analysis_metrics.json`, `data/processed/account_scoring_model_output.csv`, validation summary.
- Revenue quality / retention / risk chart sections: governed chart artifacts in `outputs/charts/*.png`.
- Account risk table and filter diagnostics: `data/processed/account_scoring_model_output.csv` + joined dimensions.
- Portfolio manager view: `data/processed/account_manager_summary.csv`, `data/raw/account_managers.csv`.
- Scenario section: `data/processed/mrr_scenario_table.csv`, `scenario_mrr_trajectories.csv`, `commercial_risk_impact_estimates.csv`.

## Performance Decisions
- Removed heavy per-customer monthly payload (`monthly_facts`) from dashboard runtime path.
- Shifted official KPI computation to backend and embedded static governed KPI values.
- Kept frontend computations limited to filter-driven account slicing and table operations.
- Embedded pre-rendered chart artifacts to avoid frontend chart recomputation and improve reliability.
- Added post-validation dashboard refresh so displayed readiness tier matches latest validation run.

## Reliability / QA Guarantees
- Dashboard payload contract is enforced in `tests/test_artifact_contracts.py`.
- Validation checks (`run_full_project_validation.py`) include dashboard feed integrity and cross-output metric reconciliation.
- Gate enforcement can block release by governance tier using `--min-readiness-tier`.

## Update Workflow
1. Run full pipeline:
   ```bash
   python3 src/pipeline/run_project_pipeline.py --base-dir . --seed 42
   ```
2. Validate strict gate:
   ```bash
   python3 src/validation/check_validation_gate.py \
     --summary-path reports/formal_validation_summary.json \
     --max-warn 0 --max-fail 0 --max-high-severity 0 --max-critical-severity 0 \
     --min-readiness-tier "technically valid"
   ```
3. Publish only `outputs/dashboard/executive_dashboard.html` generated by this path.
